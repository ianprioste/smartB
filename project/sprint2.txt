  CONFIG - MODELS (6)

           └─ List active models only

           └─ Update model (name, sizes, order)

    DELETE /config/models/{code}
           └─ Soft delete (set is_active = false)

  CONFIG - COLORS (6)
    POST   /config/colors
           └─ Create new color (or reactivate soft-deleted)
           Input: {code, name}

    GET    /config/colors
           └─ List active colors
           Query: ?all=true (include inactive)

    GET    /config/colors/{code}
           └─ Get color by code

    PUT    /config/colors/{code}
           └─ Update color

    DELETE /config/colors/{code}
           └─ Soft delete

  CONFIG - TEMPLATES (4)
    POST   /config/templates
           └─ Associate Bling product to template
           Input: {model_code, template_kind, bling_product_id}

    GET    /config/templates
           └─ List all templates
           Query: ?model_code=CAM (filter)

    GET    /config/templates/{id}
           └─ Get template details

    DELETE /config/templates/{id}
           └─ Remove association

  BLING - PRODUCTS (2)
    GET    /bling/products/search
           └─ Search Bling products
           Query: ?q=CAMLISA&page=1&limit=10
           Auto-detects SKU vs name

    GET    /bling/products/{product_id}
           └─ Get product details

🗄️ DATABASE SCHEMA (SPRINT 2)
═════════════════════════════════════════════════════════════════════════════

MODELS TABLE:
  id                  UUID, PK
  tenant_id           UUID, FK → tenants
  code                VARCHAR(50), indexed, unique with tenant_id
  name                VARCHAR(255)
  allowed_sizes       JSON (array of strings)
  size_order          JSON (array of strings, subset of allowed_sizes)
  is_active           BOOLEAN, default=True
  created_at          TIMESTAMP
  updated_at          TIMESTAMP

COLORS TABLE:
  id                  UUID, PK
  tenant_id           UUID, FK → tenants
  code                VARCHAR(50), indexed, unique with tenant_id
  name                VARCHAR(255)
  is_active           BOOLEAN, default=True
  created_at          TIMESTAMP
  updated_at          TIMESTAMP

MODEL_TEMPLATES TABLE:
  id                  UUID, PK
  tenant_id           UUID, FK → tenants
  model_code          VARCHAR(50), indexed
  template_kind       VARCHAR(50) (BASE_PLAIN, STAMP, PARENT_PRINTED, VARIATION_PRINTED)
  bling_product_id    INTEGER
  bling_product_sku   VARCHAR(255)
  bling_product_name  VARCHAR(255)
  is_active           BOOLEAN, default=True
  created_at          TIMESTAMP
  updated_at          TIMESTAMP
  Unique: (tenant_id, model_code, template_kind)

🛠️ RECURSOS AVANÇADOS
═════════════════════════════════════════════════════════════════════════════

1. SOFT DELETE + REACTIVATION
   ├─ Delete sets is_active = False (não apaga dados)
   ├─ POST com mesmo code reativa e atualiza campos
   ├─ Endpoint ?all=true lista até inativados
   └─ Preserva histórico de mudanças

2. DRAG-AND-DROP REORDERING
   ├─ Chips of sizes are draggable
   ├─ Drag to reorder visually
   ├─ size_order automatically synced
   ├─ Validates (size_order must be subset of allowed_sizes)
   └─ HTML5 native API (no dependencies)

3. INTELLIGENT PRODUCT SEARCH
   ├─ Heuristic: short uppercase no-space → SKU search (codigos[] parameter)
   ├─ Otherwise → name search (nome parameter)
   ├─ Auto-refresh OAuth2 tokens from DB
   ├─ 404 immediate stop (no retry on not found)
   └─ Portuguese error messages

4. AUTOMATIC TOKEN MANAGEMENT
   ├─ Fetch token from DB at request time
   ├─ Check expiration (5-min buffer)
   ├─ Auto-refresh if expired
   ├─ Save new token back to DB via callback
   └─ All transparent to user

5. VALIDATION LAYERS
   ├─ Pydantic validation (request schemas)
   ├─ Backend field validators (no duplicates in allowed_sizes, size_order subset)
   ├─ Database unique constraints (tenant_id + code)
   └─ Frontend form validation

6. ERROR HANDLING
   ├─ 404 → "Nenhum produto encontrado..."
   ├─ 401 → "Erro de autenticação..."
   ├─ 429 → "Limite de requisições..."
   ├─ 409 → "Modelo/Cor já existe..."
   ├─ 422 → Validation error with details
   └─ All returned as JSON with error code + message

═════════════════════════════════════════════════════════════════════════════
🔄 KEY WORKFLOWS
═════════════════════════════════════════════════════════════════════════════

CRIAR MODELO:
  1. User clica "Novo Modelo"
  2. Form aparece com inputs (code, name)
  3. Adiciona tamanhos (P, M, G) e arrasta para reordenar
  4. Clica "Salvar"
  5. POST /config/models → Backend cria
  6. Tabela atualiza com novo modelo

EDITAR MODELO:
  1. User clica "Editar" em um modelo
  2. Form preenche com dados atuais
  3. User remove um tamanho (M)
  4. size_order atualiza automaticamente (P, G)
  5. Clica "Salvar"
  6. PUT /config/models/{code} com novo allowed_sizes e size_order
  7. Backend atualiza
  8. Tabela atualiza

EXCLUIR MODELO:
  1. User clica "Excluir"
  2. Modal de confirmação aparece
  3. User clica "Excluir" no modal
  4. DELETE /config/models/{code}
  5. Backend soft deletes (is_active = False)
  6. Tabela atualiza (modelo desaparece da lista)

REATIVAR MODELO DELETADO:
  1. User tenta criar novo modelo com code "BL"
  2. No backend, encontra registro inativo com code="BL"
  3. Reativa: is_active = True
  4. Mantém name antigo: "Blusa"
  5. Atualiza: allowed_sizes = [P, M, G, GG]
  6. POST retorna 201 (reativado)
  7. Modelo reaparece na lista

BUSCAR E ASSOCIAR PRODUTO:
  1. User abre Templates
  2. Seleciona modelo "CAM - Camiseta"
  3. Escolhe template kind "BASE_PLAIN"
  4. Digita "CAMLISA" (SKU)
  5. Clica "Buscar no Bling"
  6. GET /bling/products/search?q=CAMLISA
  7. Backend detecta SKU (uppercase) → usa codigos[]
  8. Results aparecem
  9. Clica "Usar este Produto"
  10. POST /config/templates → Salva associação
  11. Aparece em "Templates Configurados"
